Difference Between Absolute Path and Relative Path

Feature Absolute Path Relative Path
Definition Complete path from the root directory to the target file Path relative to the current working directory
Starts From Root directory (/ or C:\) Current directory (.) or parent directory (..)
Consistency Always points to the same location Depends on current directory
Example (Linux) /home/user/docs/file.txt docs/file.txt
Example (Windows) C:\Users\Aman\Documents\file.txt Documents\file.txt
Use Case Accessing fixed system files Project structure portability
Clarity Clear and unambiguous Can be confusing if directory changes
Preferred When Exact location is known and fixed Within project or shared codebases
